physics engine

I was making this physics engine using roblox multithreading and eventually gave up on it. I was gonna use it for the game knockout thats right below this box but the lag was too much and the network cost was also pretty expensive. Overall i learned quite a bit about roblox multithreading, but I struggled finding the contact manifold, its pretty simple in 2d but as soon as you add that 3rd dimension everything starts burning. Thats mainly why i quit on this after spending like 2 weeks trying to figure out the contact manifold.

knockout

Game i was working on called knockout. This game was heavily inspired by gangbeasts if you know what that is. Altogether it took me like 2 months to get a playable and polishedish version. Mainly because this was my first like complex project and it involved a LOT of scripting. It was fun to work on but since all of the character controller code was held server side the input delay was like 100ms on average with interpolation. There was really no fix to this as client prediction would be almost impossible to implement. I mainly stopped working on this after I figured out that just 1 character was already sending 100kbs of data which is twice robloxs limit. This game is also mainly where I learned how to create effective and good lookingish ui.
